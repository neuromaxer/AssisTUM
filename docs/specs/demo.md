## Demo

Nico is TUM CS student

Today is Sunday and Nico wants to plan his next week

Agentum will help him with the planning

**Schedule mandatory stuff:**

_Nico types "Let's plan the next week"_

I) Agentum checks

1. Lecture & seminar times
2. Inbox for todos
3. Moodle
   1. What Nico needs to prep for seminal/lecture
   2. Assignment deadlines
4. Student club events (e.g. TUM AI)
5. Cantene for menu for the week

II) Agent puts

1. Events into calendar
2. TODOs into tasks
3. Brainstorm/reply in natural language

_Add ad-hoc personal events that agent is not aware of_

III) Agent surfaces overlapping events

**Additional planning step:**

_Nico asks agent to help plan study sessions based on assignment deadlines_

IV) Agent

1. Asks questions about workload
2. Suggests a plan
3. Puts a plan on the calendar

## UI

Should it be just one screen with

1. Left panel - tasks in chronological order of deadlines
2. Central panel - Calendar
3. Right panel - agent chat

## Connected Services

1. Lectures - TUM online (lectures list), TUM Calendar for times
2. Inbox - TUM email
3. Assignments - Moodle
4. Student club events - list of club URLs
5. Menu - cantene menu
